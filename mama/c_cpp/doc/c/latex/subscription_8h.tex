Applications may set and retireve per-\/topic closures using \hyperlink{subscription_8h_a4cb8fd940beb5511a194c02249c709a0}{mamaSubscription\_\-setItemClosure()} however this method is only guaranteed to set the correct closure if called while in the msg callback. The topicClosure argument to the callback is NULL prior to an application setting the closure for the topic.

The topic must be a valid wildcard topic for the underlying middleware.

Applications may set and retireve per-\/topic closures using \hyperlink{subscription_8h_a4cb8fd940beb5511a194c02249c709a0}{mamaSubscription\_\-setItemClosure()} however this method is only guaranteed to set the correct closure if called while in the msg callback. It will be the topicClosure argument for subsequent callback invocations for the current topic. The topicClosure argument to the callback is NULL prior to an application setting the closure for the topic.
